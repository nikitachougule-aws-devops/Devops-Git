## 🔀 Branching

A **branch** is a separate copy of your project where you can work without affecting the main project.
- main is the default branch.
- Developers create new branches to add features or fix bugs.
- Changes in one branch do not affect other branches until they are merged.
---

## 🔀 Merging

- Merging is the process of combining changes from one branch into another.
- After completing work on a branch, the code is tested and then merged into the main branch.
- This allows the new feature or bug fix to become part of the main project.
- Merging helps keep all project changes in one place.
--- 

### Simple Workflow
```bash
1. Create a branch
        ↓
2. Make changes
        ↓
3. Commit changes
        ↓
4. Switch to main
        ↓
5. Merge the branch
        ↓
6. Delete the branch
```
