## Pull Request (PR) 

- It  is a request to merge changes from one branch into another, usually from a feature branch into the main branch.
- It allows team members to review, discuss, and approve the code before it is merged.
---

## Why Use a Pull Request?

- Reviews code before merging.
- Finds bugs and mistakes early.
- Allows team members to suggest improvements.
- Ensures only approved code is merged into the main branch.

## Workflow
- Create a new branch.
- Make changes and commit them.
- Push the branch to the remote repository.
- Create a Pull Request.
- Team members review the code.
- Make changes if requested.
- Approve and merge the Pull Request.
- Delete the feature branch (optional).

**Create → Code → Push → PR → Review → Fix → Approve → Merge → Delete branch**

---

## Commands Used

- Create a new branch:
```bash
git switch -c feature-login
```
- Commit changes:
```bash
git add .
git commit -m "Add login feature"
```
- Push the branch:
```bash
git push -u origin feature-login
```
- After pushing, open GitHub (or another Git hosting service) and click New Pull Request to create the PR.
