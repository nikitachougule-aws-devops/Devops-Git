## Git Workflow

A **Git workflow** is the step-by-step process of tracking, saving, and sharing changes in a Git repository.

---

## Git Workflow Components

Git has four main areas:

1. Working Directory
2. Staging Area
3. Local Repository
4. Remote Repository

```text
Working Directory
        │
        │ git add
        ▼
Staging Area
        │
        │ git commit
        ▼
Local Repository
        │
        │ git push
        ▼
Remote Repository (GitHub)
```

---

### 1. Working Directory

The **Working Directory** is where you create, edit, and delete files.

Example:

* Create a new file.
* Modify existing code.
* Delete unwanted files.

Git notices these changes but does not save them automatically.

Check the status:

```bash
git status
```

---

### 2. Staging Area

The **Staging Area** is a temporary area where you select the changes you want to include in the next commit.

Add one file:

```bash
git add index.html
```

Add all files:

```bash
git add .
```

---

### 3. Local Repository

The **Local Repository** stores all committed changes on your computer.

Save staged changes:

```bash
git commit -m "Add login page"
```

Each commit creates a snapshot of your project.

---

### 4. Remote Repository

A **Remote Repository** is an online copy of your project hosted on platforms like GitHub.

Upload commits:

```bash
git push origin main
```

Download latest changes:

```bash
git pull origin main
```

---

## Complete Git Workflow

### Step 1: Create a Repository

```bash
git init
```

---

### Step 2: Check Status

```bash
git status
```

---

### Step 3: Add Files

```bash
git add .
```

---

### Step 4: Commit Changes

```bash
git commit -m "Initial commit"
```

---

### Step 5: Connect to GitHub

```bash
git remote add origin <repository-url>
```

---

### Step 6: Push Changes

```bash
git push -u origin main
```

---

### Step 7: Pull Latest Changes

```bash
git pull origin main
```

## Example 

```bash
git init
git status
git add .
git commit -m "Initial commit"
git remote add origin <repository-url>
git push -u origin main
```

After making changes:

```bash
git status
git add .
git commit -m "Update project"
git push
```

---

## Key Points

* **Working Directory** is where you edit files.
* **Staging Area** stores selected changes before committing.
* **Local Repository** stores commits on your computer.
* **Remote Repository** stores the project online for collaboration.
  
* A typical Git workflow is:

```text
Edit → Add → Commit → Push → Pull
```
