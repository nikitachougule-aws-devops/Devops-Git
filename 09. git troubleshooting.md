## Git Troubleshooting

## 1. You committed to the wrong branch. What will you do?

**Answer:**
Create the correct branch and move the commit there. If the commit has already been pushed, use `git revert` instead of `git reset`.

---

## 2. You accidentally deleted a branch. Can you recover it?

**Answer:**
Yes. Use `git reflog` to find the last commit, then recreate the branch.

```bash
git reflog
git switch -c recovered-branch <commit-hash>
```

---

## 3. How do you recover deleted commits?

**Answer:**
Use `git reflog` to find the commit hash, then restore it.

```bash
git reflog
git reset --hard <commit-hash>
```

---

## 4. What is a merge conflict?

**Answer:**
A merge conflict occurs when Git cannot automatically merge changes because the same part of a file was modified in different branches.

**Steps:**

```bash
git merge feature
```

Open the conflicted file and remove the conflict markers:

```text
<<<<<<< HEAD
Current code
=======
Incoming code
>>>>>>> feature
```

Then:

```bash
git add .
git commit
```

---

## 5. You accidentally committed a password or API key. What will you do?

**Answer:**
Remove the secret from the code, change (rotate) the password or API key immediately, and update the Git history if required.

---

## 6. What is a Detached HEAD?

**Answer:**
A Detached HEAD occurs when HEAD points to a commit instead of a branch.

**Fix:**

```bash
git switch main
```

To save your work:

```bash
git switch -c new-branch
```

---

## 7. How do you undo the last commit but keep the changes?

```bash
git reset --soft HEAD~1
```

---

## 8. How do you undo a pushed commit?

```bash
git revert <commit-hash>
```

This creates a new commit that reverses the previous one.

---

## 9. You have unfinished work but need to switch branches. What will you do?

**Answer:**
Save your work using stash.

```bash
git stash
git switch <branch-name>
git stash pop
```

---

## 10. You accidentally deleted an important file.

**Answer:**
Restore it from the last commit.

```bash
git restore <file>
```

---

## 11. Your local branch is behind the remote branch.

**Answer:**
Pull the latest changes.

```bash
git pull origin main
```

---

## 12. You pushed the wrong commit.

**Answer:**
Use `git revert` if the commit has already been shared. Use `git reset` only if it has not been shared.

---

## 13. How do you find who changed a line of code?

```bash
git blame <file>
```

---

## 14. How do you compare two commits?

```bash
git diff <commit1> <commit2>
```

---

## 15. How do you find which commit introduced a bug?

```bash
git bisect
```

Git automatically helps locate the commit that introduced the bug.

---

## 16. Wrong Commit Message

**Answer:**
Edit the last commit message.

```bash
git commit --amend -m "New commit message"
```

---

## 17. Accidentally Added the Wrong File

**Answer:**
Remove the file from the staging area.

```bash
git restore --staged <file>
```

---

## 18. Repository Has Untracked Files

**Answer:**
Preview the files first:

```bash
git clean -n
```

Delete them:

```bash
git clean -f
```

---

## 19. Push Rejected Because Remote Has New Changes

**Answer:**
Pull the latest changes and then push again.

```bash
git pull origin main
git push
```

---

# Quick Revision

| Problem                    | Solution                                                |
| -------------------------- | ------------------------------------------------------- |
| Committed to wrong branch  | Create the correct branch or use `git cherry-pick`      |
| Deleted branch             | `git reflog`                                            |
| Lost commit                | `git reflog` + `git reset`                              |
| Merge conflict             | Resolve conflicts → `git add` → `git commit`            |
| Password/API key committed | Remove it, rotate the secret, clean history if required |
| Detached HEAD              | `git switch main` or `git switch -c new-branch`         |
| Undo last commit           | `git reset --soft HEAD~1`                               |
| Undo pushed commit         | `git revert`                                            |
| Unfinished work            | `git stash`                                             |
| Deleted file               | `git restore <file>`                                    |
| Local branch behind        | `git pull`                                              |
| Wrong commit message       | `git commit --amend`                                    |
| Wrong staged file          | `git restore --staged <file>`                           |
| Untracked files            | `git clean`                                             |
| Push rejected              | `git pull` then `git push`                              |
| Find who changed a line    | `git blame`                                             |
| Compare commits            | `git diff <commit1> <commit2>`                          |
| Find bug commit            | `git bisect`                                            |
