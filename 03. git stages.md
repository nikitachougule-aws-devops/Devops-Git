## Git Stages

Git tracks changes through different **stages** before they are permanently saved.

There are **four Git stages**:

1. Working Directory
2. Staging Area
3. Local Repository
4. Remote Repository

These stages help developers organize, review, and share changes.

---

## Git Stages Diagram

```text
                git add            git commit          git push
Working Directory ─────────▶ Staging Area ─────────▶ Local Repository ─────────▶ Remote Repository
      ▲                                                                                 │
      └────────────────────────────── git pull / git fetch ─────────────────────────────┘
```

---

### 1. Working Directory

The **Working Directory** is where you create, edit, or delete files.

Examples:

* Create a new file.
* Edit an existing file.
* Delete a file.

Git detects these changes but does not save them automatically.

Check the status:

```bash
git status
```

---

### 2. Staging Area (Index)

The **Staging Area** is a temporary place where you select the changes to include in the next commit.

Add a specific file:

```bash
git add index.html
```

Add all files:

```bash
git add .
```

You can review staged files before committing.

---

### 3. Local Repository

The **Local Repository** stores committed changes on your computer.

Commit staged changes:

```bash
git commit -m "Add login page"
```

Each commit creates a snapshot of the project that you can return to later.

---

### 4. Remote Repository

A **Remote Repository** is an online copy of your project hosted on platforms such as GitHub.

Upload commits:

```bash
git push origin main
```

Download the latest changes:

```bash
git pull origin main
```

---

## Git Stage Flow

```text
Create or Edit File
        │
        ▼
Working Directory
        │
   git add
        ▼
Staging Area
        │
 git commit
        ▼
Local Repository
        │
  git push
        ▼
Remote Repository
```

---

## Example

Create a file:

```bash
echo "Hello Git" > hello.txt
```

Check the status:

```bash
git status
```

Stage the file:

```bash
git add hello.txt
```

Commit the file:

```bash
git commit -m "Add hello.txt"
```

Push the commit:

```bash
git push origin main
```

---

## Common Commands for Each Stage

| Stage             | Command                         |
| ----------------- | ------------------------------- |
| Working Directory | `git status`                    |
| Staging Area      | `git add <file>` or `git add .` |
| Local Repository  | `git commit -m "message"`       |
| Remote Repository | `git push origin main`          |
| Download Changes  | `git pull origin main`          |

---

# Key Points

* **Working Directory** is where you make changes.
* **Staging Area** is where you prepare changes for a commit.
* **Local Repository** stores commits on your computer.
* **Remote Repository** stores the project online for collaboration.
  
* The Git stage flow is:

```text
Working Directory → Staging Area → Local Repository → Remote Repository
```
