## Git Commands

## Setup and Configuration

* `git config --global user.name "Name"` → Sets your Git username.
* `git config --global user.email "email@example.com"` → Sets your Git email.
* `git config --list` → Displays all Git configuration settings.

---

## Repository Management

* `git init` → Creates a new Git repository.
* `git clone <repository-url>` → Downloads (clones) an existing repository.

---

## Basic Workflow

* `git status` → Shows the current status of files.
* `git add <file>` → Adds a specific file to the staging area.
* `git add .` → Adds all changed files to the staging area.
* `git commit -m "message"` → Saves staged changes.
* `git commit -am "message"` → Stages and commits tracked files only.

---

## Viewing History

* `git log` → Shows complete commit history.
* `git log --oneline` → Shows a short commit history.
* `git log --graph` → Shows commit history as a graph.
* `git show <commit>` → Displays details of a specific commit.
* `git diff` → Shows unstaged changes.
* `git diff --staged` → Shows staged changes.
* `git blame <file>` → Shows who last modified each line of a file.

---

## Branching

* `git branch` → Lists all local branches.
* `git branch feature-login` → Creates a new branch.
* `git branch -d <branch-name>` → Deletes a merged branch.
* `git branch -D <branch-name>` → Force deletes an unmerged branch.
* `git switch <branch-name>` → Switches to an existing branch.
* `git switch -c feature-login` → Creates and switches to a new branch.
* `git checkout feature-login` → Switches to an existing branch.
* `git checkout -b feature-login` → Creates and switches to a new branch.

---

## Merging

* `git switch main` → Switches to the `main` branch.
* `git merge feature-login` → Merges the `feature-login` branch into the current branch.
* `git branch -d feature-login` → Deletes the merged branch.
* `git branch -D feature-login` → Force deletes an unmerged branch.

---

## Remote Repositories

* `git remote add origin <repository-url>` → Connects the local repository to a remote repository.
* `git remote -v` → Lists configured remote repositories.
* `git fetch` → Downloads changes from the remote repository.
* `git pull` → Downloads and merges remote changes.
* `git push` → Uploads local commits to the remote repository.
* `git push -u origin main` → Pushes the `main` branch and sets it as the upstream branch.

---

## Undoing Changes

* `git restore <file>` → Discards changes in a file.
* `git restore --staged <file>` → Removes a file from the staging area.
* `git reset <file>` → Unstages a file.
* `git reset --soft HEAD~1` → Removes the last commit but keeps the changes staged.
* `git reset --hard HEAD~1` → Removes the last commit and deletes the changes.
* `git revert <commit>` → Creates a new commit that reverses a previous commit.

---

## Stashing

* `git stash` → Temporarily saves uncommitted changes.
* `git stash list` → Lists all saved stashes.
* `git stash apply` → Applies the latest stash without deleting it.
* `git stash pop` → Applies the latest stash and removes it.
* `git stash drop` → Deletes a specific stash.

---

## Tags

* `git tag` → Lists all tags.
* `git tag <tag-name>` → Creates a lightweight tag.
* `git tag -a <tag-name> -m "message"` → Creates an annotated tag.
* `git push origin <tag-name>` → Pushes a tag to the remote repository.

---

## Searching

* `git grep "text"` → Searches for text in project files.
* `git log -S "keyword"` → Searches commit history for changes containing a keyword.

---

## Cleaning

* `git clean -n` → Previews files that will be deleted.
* `git clean -f` → Deletes untracked files.

---

## Advanced Commands

* `git cherry-pick <commit>` → Applies a specific commit to the current branch.
* `git reflog` → Shows the history of `HEAD` and branch references.
* `git bisect` → Finds the commit that introduced a bug.
* `git archive` → Creates an archive of the repository.
* `git submodule add <repository-url>` → Adds a Git submodule.

---

## Useful Shortcuts

* `git commit --amend` → Modifies the last commit.
* `git push --force` → Force pushes changes to the remote repository.
* `git push --force-with-lease` → Safely force pushes without overwriting others' work.
* `git diff HEAD~1 HEAD` → Compares the last two commits.
